# V19 二次功能补充开发说明

整理日期：2026-07-12  
依据文件：`待补充更新需求V19-平台功能更新V19对照矩阵.html`

## 一、已完善功能

### 1. 新增领导角色

- 新增演示账号：`周明远 / 领导 · 决策查看`。
- 领导角色默认首页为可视化驾驶舱。
- 领导侧边栏仅开放：
  - 可视化驾驶舱；
  - 项目台账；
  - 成果转化台账。
- 项目台账和成果转化台账展示领导只读标识。
- 项目台账导出接口对领导角色返回 403，默认不开放全量导出。
- 配置中心角色权限矩阵已增加“领导”列。

### 2. 后评价暂缓/删除

- 主导航移除“后评价管理”。
- `/post-evals` 旧路由重定向至协作评价页。
- 驾驶舱、预警中心、审批中心接口过滤后评价相关数据。
- 项目详情生命周期图调整为：
  - 项目申报；
  - 立项备案；
  - 实施阶段；
  - 项目验收；
  - 成果转化。
- 项目详情归档阶段列表移除“后评价”。
- 数据库表结构仍保留 `post_evals`，作为后续阶段接口预留。

### 3. 项目台账 V19 字段口径补充

- 后端为项目台账新增 V19 派生字段组 `v19`，覆盖：
  - 一级专业、二级专业；
  - 管理单位、需求单位、责任单位；
  - 立项年月、结束年月、项目周期；
  - 牵头单位/主要工作内容；
  - 国拨经费、自筹经费、商飞内部单位经费；
  - 累计支出、结题实际执行经费、执行率；
  - 交付物摘要、协作评价摘要、成果转化摘要。
- 项目台账页面新增/调整列：
  - 专业/单位；
  - 项目周期；
  - 经费拆分；
  - 交付/协作；
  - 成果转化。
- 项目台账 Excel 导出改为 V19 口径，并附加“V19字段口径”工作表。

### 4. 可视化驾驶舱 V19 指标优化

- 新增总执行率 KPI，区分年度执行率与项目总执行率。
- 新增国拨/自筹/商飞内部经费构成指标。
- 新增“承担单位 × 项目层级承担矩阵”，展示各单位国家级、地方级、公司级项目承担情况。
- 新增计划管理状态区，展示红/黄/蓝/绿计划数量、办结率、CMOS 同步状态。
- 新增经费构成图，展示总经费、国拨、自筹、商飞内部经费。
- 新增型号转化分布图，按成果包向型号转化对象统计。
- 渠道分布标题调整为“层级关联下的渠道项目分布”，体现层级与渠道关联。

### 5. 立项查重工具入口

- 项目申报页在“项目名称”处新增“项目查重”按钮。
- 新增后端接口：`POST /api/project-duplicates`。
- 当前实现基础字段查重：
  - 项目名称；
  - 项目编号；
  - 项目渠道；
  - 项目级别；
  - 责任单位；
  - 项目负责人；
  - 关键词。
- 查重结果展示：
  - 疑似重复项目；
  - 相似度；
  - 命中字段；
  - 项目状态；
  - 处理建议。
- 每次查重写入审计日志，便于后续追溯。

### 6. 成果转化独立业务板块

- 新增独立页面：`/transformations`。
- 新增导航入口：
  - 管理团队：成果转化台账；
  - 领导：成果转化台账；
  - 项目详情中仍保留项目级成果转化页签。
- 新增后端接口：`GET /api/transformations`。
- 页面能力：
  - 成果包总数、向型号转化、向市场转化、已完成、逾期成果统计；
  - 按状态、方式、单位、关键字筛选；
  - 成果包列表展示成果编号、关联项目、转化方式、转化形式、状态、计划时间、交付物数量；
  - 展示型号/应用对象转化分布；
  - 领导角色只读视图。

### 7. 表单过渡工具

- 新增独立页面：`/transition-tool`。
- 管理团队和超级管理员导航可访问。
- 新增后端接口：
  - `GET /api/transition-tool`：获取字段口径、分表、总表汇总、校验结果；
  - `POST /api/transition-tool/import-demo`：按现有项目数据生成演示批量导入数据；
  - `POST /api/transition-tool/records`：维护分表记录并自动汇总；
  - `GET /api/transition-tool/export.xlsx`：一键导出总表、专项分表、字段口径、校验问题。
- 页面能力：
  - 按国家级、地方级、公司级拆分专项分表；
  - 自动汇总总表；
  - 展示校验通过、待修正、重复项目；
  - 支持编辑分表记录；
  - 支持一键导出 Excel。

## 二、已预留但待补充接口

### 1. 查重高级算法接口

当前已实现基础字段查重。后续可接入：

- 历史项目知识库；
- 语义相似度模型；
- 关键词/摘要向量检索；
- 强阻断阈值规则。

预留位置：`POST /api/project-duplicates`。

### 2. 表单过渡工具真实 Excel 批量上传

当前已实现演示导入、分表维护、自动汇总和导出。真实 Excel 上传仍依赖业务侧提供：

- 各专项真实表格；
- 填写说明；
- 下拉菜单；
- 汇总总表模板；
- 字段冲突处理规则。

预留位置：

- `TRANSITION_FIELDS` 字段口径；
- `/api/transition-tool` 接口族；
- 表单过渡工具页面。

### 3. 内网部署与多人访问

当前工具已按“单台内网机器集中服务、多主管浏览器访问”设计。实际部署需确认：

- 内网机 IP；
- 端口开放；
- 运行环境安装权限；
- 安全审查；
- 数据备份方式；
- 是否允许安装 Node.js/SQLite 依赖。

### 4. 经费总执行率最终公式

当前总执行率采用：

`累计支出 / 项目总经费`

若业务侧改为“累计支出 / 可执行经费”或其他口径，只需调整驾驶舱接口中的 `fundStructure.totalExecRate` 和 `kpis.totalExecRate`。

### 5. 后评价后续阶段

后评价本轮已隐藏并过滤，但数据库表结构、字段类型和旧数据结构保留，可在后续版本重新开放。

## 三、待业务侧确认事项

- 领导角色是否允许导出项目台账；当前默认不允许。
- 查重结果是否需要强制阻断提交；当前仅提示并留痕。
- 表单过渡工具真实字段口径、下拉枚举和各专项分表差异。
- 表单过渡工具部署形态：内网单机服务还是单人电脑本地使用。
- 经费总执行率最终公式。
- 成果转化中“型号/应用对象”是否有统一字典。

## 四、验证结果

- 服务端语法检查：通过。
  - 命令：`node --check server/src/api.js`
  - 命令：`node --check server/src/seed.js`
- 前端生产构建：通过。
  - 命令：`npm.cmd run build`
- 说明：
  - 本机 `server` 依赖安装未完全完成，原因是当前 Node.js 24 缺少 `better-sqlite3` 对应预编译包，且本机未安装 Visual Studio C++ 构建工具，无法编译原生依赖。
  - 前端构建不依赖 server 原生依赖，已完成构建验证。
  - 若需完整运行 server，建议使用项目原定 Docker 环境，或切换到有 `better-sqlite3` 预编译支持的 LTS Node 版本并安装对应依赖。
