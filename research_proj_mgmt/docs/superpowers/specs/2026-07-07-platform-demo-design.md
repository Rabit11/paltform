# 科研项目信息化管理平台 — 交互演示版 设计规格

日期：2026-07-07 · 状态：已批准（用户授权全程自主执行）
依据：《科研项目信息化管理平台需求V18.docx》全文精读 + 内嵌功能架构图/流程图/看板样例解析

## 1. 目标与定位

为客户构建一个**可交互、可演示、带真实后端数据库**的平台演示版（Demo），完整呈现需求文档的六大业务域与五类角色视角。演示版以"讲清楚业务闭环 + 视觉惊艳"为第一目标，不追求生产级鉴权/集成（CMOS、经费系统等外部对接以"已同步"的仿真数据呈现）。

先前多轮产出（HTML 幻灯片、空壳 backend）全部废弃，本次从零重建。

## 2. 业务域映射（需求 → 演示功能）

| 需求模块 | 演示实现 |
|---|---|

| 项目台账 | 多条件筛选 + 列配置 + CSV导出 + 点击穿透详情 |
| 可视化看板（总部专属） | "驾驶舱"页：KPI带 + 六维图表 + 风险榜单 + 维度筛选 + 点击溯源 |
| 立项阶段（差异化材料+审签流） | 申报向导：选渠道 → 自适应材料清单 → 提交生成审批流实例 |
| 实施阶段（基本信息/里程碑/计划/经费/评估/变更） | 里程碑填报闭环销项、CMOS 计划同步与办结、双轨经费、变更两类入口 |

| 后评价（>1亿, 3年内） | 后评价列表 + 流程状态 |
| 预警机制（站内+邮箱+蓝信） | 预警中心 + 顶栏铃铛（推送渠道以标签示意） |

## 4. 视觉设计语言（克制·严谨·高科技）

- **基调**：深空海军蓝暗色系（#0B1120 底 / #111A2E、#16213A 面），拒绝纯黑；文字用暖灰白 #E2E8F0，弱文字 #94A3B8。
- **色纪律**：唯一品牌强调色 `青蓝 #38BDF8`（交互/选中/数据主系列）；四色状态（红#F87171 黄#FBBF24 蓝#38BDF8 绿#34D399）为仅有的语义色。杜绝彩虹图表。
- **排版**：中文系统栈（Microsoft YaHei UI/PingFang SC）+ 数字一律 tabular-nums 等宽（Consolas/JetBrains Mono 栈），Bloomberg 终端式数据密度。
- **质感**：1px 发丝边框 rgba(148,163,184,.12)、蓝图网格纹理、极轻背景辉光；不用重玻璃拟态、不用大面积渐变。
- **图形资产（全部手工 SVG）**：平台徽标（翼形+轨道）、蓝图风飞机三视图线稿、雷达扫描动画、全生命周期六阶段示意图、约20枚 1.5px 描边域图标（立项/里程碑/经费/验收/成果/评审/预警/变更/协作/后评价/字典/审计…），generic 图标用 lucide-react，同一描边语言。
- **动效**：150–250ms，入场 stagger ≤50ms/项，KPI 数字 count-up，驾驶舱雷达扫描为唯一持续动画；尊重 prefers-reduced-motion。
- **布局**：驾驶舱"主-辅"分区（核心 KPI 上左，风险榜单右缘）；工作页为 240px 侧栏 + 顶栏壳层；桌面优先（演示场景为投屏/大屏），最低适配 1280px。

## 5. 技术架构

```
platform/
  server/   Node 22 + Express + better-sqlite3（SQLite 文件库）
            db.js(建表) seed.js(造数) domain.js(四色/聚合) api.js(REST) index.js
  web/      Vite + React 18 + TypeScript + Tailwind CSS v4 + ECharts + react-router
            自定义暗色 ECharts 主题；/api 开发代理；构建后由 server 静态托管
```

- 单命令演示：`npm run demo`（种子 → 构建 → 8787 端口一体启动）；开发 `npm run dev`。
- 所有业务变更（审批、申报、销项、办结、拨付）走 REST 写库并记审计日志，演示中可当场展示数据落库。
- 四色状态由后端按 `dueDate - today` 实时计算，演示任意日期打开颜色语义仍正确。

## 6. 种子数据（编造但拟真，锚定 2026-07）

- 6 家承担单位（上飞院、上飞公司、北研中心、客服公司、试飞中心、基础能力中心）；13 条渠道全量入库。
- ~40 个航空科研项目（氢电验证机、短舱降噪、复材机翼损伤容限、SAF 适配、飞控余度、增材钛框、结冰风洞……），覆盖三级渠道、2022–2026 周期、200万–2.3亿经费；≥1 个超1亿已验收项目触发后评价。
- 每项目 3–6 个里程碑，刻意分布出 红/黄/蓝/绿 全谱；CMOS 计划、年度经费（预算/支出/核销）、交付物（专利/论文/软著/标准/样机/设备）、成果包（装机+转让案例）、协作单位五维评分（含 1 个 <60 分黑名单案例）、在途审批 10+（申报/变更/验收/办结/拨付各类型）、预警若干、审计日志。

## 7. 风险与取舍

- better-sqlite3 原生模块若安装失败 → 降级为同接口 JSON 文件存储（DAO 层隔离）。
- 不做真实登录/密码；角色即会话。不做移动端优先（演示场景为桌面投屏）。
- Excel 导出以 CSV 实现（Excel 可直接打开）。
