# 预研项目管理平台 — 需求与交付规划

**版本**：V1.0
**日期**：2026-07-07
**需求方**：黄兆辉（科技部）
**交付方**：张弛-北研中心、林宇飞、杨会冉

---

## 一、项目背景

### 1.1 当前现状
- 项目管理工作通过**线下表单**（不联网）进行
- 数据分散，无法实时监控项目进度
- 管理效率低，跨部门协作困难

### 1.2 建设目标
- 建设**预研项目管理平台**，实现项目信息在线化、实时化管理
- 服务科技部及总师层级，聚焦宏观进度监控
- 减轻基层负担，避免重复录入

### 1.3 数据起点
- **2026年**项目数据优先导入
- 历史数据逐步迁移

## 三、长期需求规划

### 3.1 系统对接需求

| 对接系统 | 需求描述 | 优先级 |
|---------|---------|-------|
| **人力系统** | SSO单点登录、人员信息同步（必须从人力系统拉取，不自建） | 🔴 高 |
| **财务系统** | 经费数据读取（仅读权限） | 🟡 中 |

| **其他系统** | 预留数据接口，以表单形式输出对接 | 🟢 低 |

### 3.2 AI能力需求

| AI功能 | 需求状态 | 说明 |
|-------|---------|------|
| 建议书智能识别 | ✅ 明确需要 | 自动提取项目目标等关键信息 |
| 查重与合规检查 | ⚠️ 探索中 | 借鉴创新股查重系统逻辑 |

> **原则**：AI能做多少是多少，过程中探索，不为了AI而AI，不增加基层负担

### 3.3 远期功能规划

| 功能模块 | 描述 | 优先级 |
|---------|------|-------|

## 七、核心决策

1. **取消独立可视化看板**：平台输出数据表单，由下游系统负责可视化
2. **人员数据来源**：必须从人力系统拉取，不自建人员档案
3. **开发策略**：先满足急切需求，预留扩展接口，复杂功能后期迭代
4. **历史数据迁移**：优先导入2026年及以后新项目，历史数据逐步迁移
5. **AI能力**：建议书智能识别为明确需求，其他AI能力视实际效果定
